# llm-wiki-starter

A ready-to-use scaffold for Andrej Karpathy's [LLM Wiki concept](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f): an LLM-maintained, Obsidian-browsed personal knowledge base that compounds over time instead of re-deriving knowledge on every query.

> *"Obsidian is the IDE. The LLM is the programmer. The wiki is the codebase."*

## What's in this repo

| File | Purpose |
|---|---|
| `BRIEF.md` | Full product brief — vision, architecture, ingest/query/lint workflows, Obsidian setup, page format standards, bootstrap sequence |
| `PRD.md` | Product requirements document — functional requirements, slash command specs, implementation phases, success metrics |
| `.claude/commands/` | Ready-to-use Claude Code slash commands: `/bootstrap`, `/ingest`, `/sync`, `/lint` |

## What this adds on top of the original Gist

- **Full PRD** with acceptance criteria per phase
- **`/sync` command** — offline workflow: drop sources into `raw/` without an active Claude session, run `/sync` on reconnect to batch-ingest everything new
- **Slash command templates** in `.claude/commands/` — copy into your vault and they work immediately
- **Schema co-evolution rules** — when and how to update `CLAUDE.md` (Claude Code's instruction file, generated by `/bootstrap` — it's the schema that governs how Claude maintains your wiki), with a `## Schema History` convention
- **Lint workflow** with a contradiction resolution protocol (investigate / resolve / accept-tension)
- **Index scaling guidance** — what to do at 50, 100, and 300 sources

## Prerequisites

**Required:**
- [Claude Code](https://claude.ai/code) — the CLI that runs the slash commands
- [Obsidian](https://obsidian.md) — free; used to browse and navigate the wiki

**Recommended:**
- Git — for version history of your wiki
- [Obsidian Web Clipper](https://obsidian.md/clipper) browser extension — clip articles directly to `raw/`
- Obsidian community plugins: **Dataview** (frontmatter queries and dynamic tables), **Obsidian Git** (auto-commit wiki changes)

## How to use

1. Create a new directory for your wiki topic
2. Copy this repo's `.claude/` directory into it — your wiki directory should end up with `.claude/commands/` containing the five command files
3. Open Claude Code in that directory
4. Run `/bootstrap [your domain description]`
5. Open the directory as an Obsidian vault

From there: drop sources into `raw/`, run `/ingest <filename>`, ask questions, run `/lint` every 5–10 sources. Sources can be markdown notes, plain-text articles, or text-extractable PDFs. Scanned image-only PDFs are not supported.
